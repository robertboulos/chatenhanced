# Task ID: 14
# Title: Implement Parameter Validation Endpoint
# Status: pending
# Dependencies: 1, 3
# Priority: low
# Description: Create an endpoint to validate generation parameters before submission.
# Details:
Create a new endpoint at `POST /api:v2/utils/validate` with the following implementation:

1. Accept the same input parameters as the generation endpoints
2. Validate all parameters according to provider-specific rules
3. Return validation results with any errors or warnings

Implement comprehensive validation for all parameters, including:
- Prompt length and content
- LoRA count and weights
- Size constraints
- Step count and guidance scale ranges
- Provider-specific limitations

The endpoint should not perform any actual generation, only validation.

# Test Strategy:
Test the endpoint with various input combinations:
- Valid inputs for different providers
- Invalid inputs (exceeding limits, wrong formats)
- Edge cases (minimum/maximum values)
- Verify that validation results are accurate
- Verify that error messages are clear and helpful
- Test performance under load
